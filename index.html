API Documentation: Dog Food Compatibility

Introduction

This API allows users to check if a certain food is safe for dogs to eat. The API utilizes a database of known safe and unsafe foods for dogs and returns a response indicating whether the requested food is safe or not.

Endpoint

The API has a single endpoint that accepts GET requests in the following format:

https://dog-food-api.onrender.com/{food_name}

Where {food_name} is the name of the food the user wants to check.

Response

The API returns a JSON object with the following properties:

  • food (string): The name of the food that was queried.
  • status (string): Indicates whether the food is safe for dogs to eat ("safe") or not ("unsafe").
  • message (string): Additional information about the food and why it is or is not safe for dogs to eat.

Example

A GET request to the endpoint https://api.dogfoodcompatibility.com/food/chicken will return the following response:

{
    "food": "chicken",
    "status": "safe",
    "message": "Chicken is a lean protein source that is safe for dogs to eat in small amounts. It should be cooked and served plain, without any seasonings or sauces."
}

Error Handling

If the API is unable to find a match for the requested food in its database, it will return a response with the following properties:

  • status: "error"
  • message: "Food not found in database"

If an invalid endpoint is requested, the API will return a response with the following properties:

  • status: "error"
  • : "Invalid endpoint"

Note

This API is for reference only and should not be used as a substitute for professional veterinary advice. Always consult a veterinarian before introducing new foods to your dog's diet.
